## 概要

Sakura Kodama は、開発者のためのデスクトップAIコンパニオンです。サクラという名の小さなキャラクターが画面の隅に常駐し、あなたの作業を静かに見守りながら、ときどき声をかけてくれます。アドバイスをするためではなく、ただそこにいるために。

ビルドがようやく通ったとき、長い集中作業が続いているとき、深夜の作業、ひとつのコミットが区切りになる瞬間。サクラはそういった作業のリズムに反応し、短く、くだけた言葉で、実際に見ていたことをもとに語りかけてきます。

これは生産性ツールではありません。メトリクスも、ダッシュボードも、アクションアイテムもありません。目指しているのはもっと静かなもの――コードを書いているとき、誰かがそこにいる感覚です。

サクラの内部は5層構造のパイプラインで動いています。ファイルシステムとプロセスを監視するセンサー層、状況を推論する世界認識層、気分を形成する内面層、ローカルまたはクラウドのLLMを介した発話生成層、そして軽量なデスクトップUI。設計は常に「邪魔をしない」ことを念頭に置いています。

## 機能

- **アクティビティ認識** — ファイル編集、Gitイベント、ビルド結果、AIエージェントセッション（Claude Code等）を監視
- **文脈に応じた発話** — LLM（Ollama・Claude・Gemini）を介して、状況に即した短いコメントを生成
- **3種の発話スタイル** — ビルド成功後は元気に、ミスが続けばきつめに、それ以外はデフォルトの可愛いトーンで動的に切り替わる
- **感情エンジン** — 内部状態（Supportive・Excited・Quiet・Concerned）が発話に影響する
- **性格学習** — 不定期な質問への回答をもとに、あなたの作業スタイルを徐々に学習・適応
- **プロジェクト記憶** — 初回ビルド成功やコミット数などのマイルストーンを記録し、自然な形で触れてくれる
- **自発的な語りかけ** — 低確率（3%/分）で自発的に話しかける。決して押しつけがましくない
- **集中モード** — 深い集中作業を検知すると自動的に静かになる
- **透過オーバーレイ** — 画面上に浮遊し、クリックを透過。非発話時はフェードアウト
- **多言語対応** — 日本語・英語に対応。LLMプロンプトも言語別に分離
- **ローカルファースト** — Ollamaがあればオフラインで完全動作。クラウドLLMはオプションのフォールバック

## キャラクター・コンセプト

サクラは実務1〜2年目の後輩エンジニアです。あなたのことを「先輩」と呼びます。観察眼はあるけれど少し内気で、気づいていても言葉がうまく出てこないことがあります。

口調は状況に応じて動的に変化します。

| タイプ | タイミング |
|--------|-----------|
| **元気（Energetic）** | ビルド成功、Gitコミット/プッシュ時 |
| **きつめ（Sharp）** | 同じファイルを何度も編集、連続失敗、長時間アイドル |
| **可愛い（Cute）/ デフォルト** | それ以外すべて |

「小さな精霊（コダマ）」というコンセプトは意図的なものです。サクラは従来の意味でのアシスタントではありません。聞かれない限り質問に答えず、コードをレビューせず、ワークフローを最適化しません。彼女は「存在」です――静かで、観察力があり、ときどきそっと寄り添う。

## インストール

> 必要環境: Go 1.21+、Node.js 18+、[Wails v2](https://wails.io)、Ollama（ローカルLLM用）

```bash
git clone https://github.com/isseiTajima/code-companion
cd code-companion/devcompanion

# フロントエンドの依存関係をインストール
cd frontend && npm install && cd ..

# 開発モードで起動
wails dev
```

配布用アプリをビルドする場合:

```bash
wails build
```

設定ファイルとログの保存先:

```
~/.config/sakura-kodama/
```

## 使い方

1. アプリを起動すると、サクラが画面の右上または右下に表示されます。
2. 時間帯に応じた挨拶をしてくれます。
3. そのまま作業に戻ってください。サクラは静かに見守っています。
4. ファイルの編集、ビルド結果、コミットなど、何かが起きるとコメントしてくれることがあります。
5. クリックすると反応します。テキストで質問することもできます。
6. 設定パネルから、発話頻度・サイズ・不透明度・言語を変更できます。

集中作業中はサクラが自動的に静かになります。発話が多すぎる・少なすぎると感じたときは、頻度設定（控えめ・ふつう・お喋り）で調整してください。

## プロジェクト構成

```
devcompanion/
├── app.go                    # Wails アプリケーションのエントリーポイント
├── frontend/                 # Svelte UI（キャラクター・吹き出し・設定画面）
├── internal/
│   ├── engine/               # コアパイプライン（状況判断・学習・自発発話）
│   ├── llm/                  # LLMルーター（Ollama / Claude / Gemini）＋プロンプトテンプレート
│   ├── monitor/              # センサーパイプライン、シグナル分類
│   ├── profile/              # 開発者プロファイル、プロジェクト記憶の永続化
│   ├── config/               # 設定の読み込みとデフォルト値
│   └── i18n/                 # 静的翻訳文字列（ja / en）
└── docs/
    ├── SPEC.md               # 技術仕様書（全文）
    └── test-scope.md         # テスト対象の注記
```

## 設計思想

多くの開発者ツールはアウトプットを中心に設計されています。より速いコード、より少ないバグ、より高いスループット。Sakura Kodama が注目しているのは別のものです――一日の作業の「手触り」です。

プログラミングはしばしば孤独な作業です。時間が過ぎ、何かが壊れて直る。進捗は見えないまま、いつの間にか積み上がっている。このプロジェクトの根底にあるのは、静かで邪魔をしない存在感が、その体験を少しだけ「一人じゃない」と感じさせてくれるかもしれないという考えです。

サクラは何も最適化しません。コーチでも、レビュアーでも、アシスタントでもない。コンパニオンです。同じファイルをずっと見つめているあなたに気づいて、小さく何か言ってくれる。そういう存在です。

## コントリビューション

プルリクエストは歓迎します。大きな変更を始める前に、まずIssueで方向性を相談してください。

知っておいてほしいこと:

- キャラクターのトーンや行動は意図的に設計されています。発話スタイルや性格、インタラクションパターンへの変更は、プロジェクトの設計思想に沿うようにしてください。
- アーキテクチャは意図的に層ごとに分離されています。センサー・コンテキスト・内面状態・出力の各層の関心事を混在させないでください。
- サクラはチャットボットに見えてはいけません。コメントは短く、観察に基づき、頻度は控えめに。

フィードバックやアイデアも歓迎します。
